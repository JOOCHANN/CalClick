# PRD: CalClick

## 목표 (one-liner)
한상차림 1장을 찍으면 국·밥·반찬을 각각 인식하고, 식약처·농진청 공식 DB에서 칼로리를 찾아, 공유 식사는 인원수로 나눠 정확히 계산한다.

## 사용자
- 다이어트·식단 관리하는 20-40대 한국인
- 외식·배달·한상차림 빈도가 높아 해외 앱으로는 입력이 고통스러운 사람
- 칼로리 입력의 마찰(검색·선택·수량 입력)을 촬영 한 번으로 줄이고 싶은 사람

## 경쟁 대비 차별점 (왜 만드는가)
1. **한상차림 다중 인식**: Cal AI·SnapCalorie·BitePal은 단일 음식 중심. CalClick은 국·밥·반찬을 동시 분리 인식.
2. **공식 DB 기반 신뢰도**: 식약처·농진청 공식 영양표 기반. 출처·수치 UI에 노출.
3. **공유 식사 UX**: 찌개/반찬 공유는 "몇 명이서" 입력 → 자동 ÷N 분할.

## MVP 핵심 기능 (Phase 0-3까지)
1. 사진 촬영/업로드 → 한상차림 **다중 음식** 인식
2. 식약처 DB 매칭 → 칼로리·탄·단·지 분해 표시 (출처 라벨)
3. 사용자 수정: 음식명 · 중량 슬라이더 · 공유 인원 토글
4. 오늘 kcal 합계 (로그인 사용자)

## v1.0 (Phase 4-6)
- Daily diary (날짜별 식단 · 끼니 분류 · 사진 갤러리)
- 프로필 · BMR/TDEE · 목표 kcal · 주/월 그래프 · 체중 로그
- PWA polish: 오프라인 촬영 큐잉 · Lighthouse PWA ≥ 90

## MVP 제외 (명시)
- 소셜/친구/챌린지
- 바코드 스캔
- 운동 기록 · 웨어러블 연동
- 레시피 제안 · AI 코칭 챗
- Push 알림
